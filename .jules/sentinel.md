## 2024-05-19 - [Missing Secure Screen Protection on Independent Entry Points]
**Vulnerability:** Independent activities like `CrashActivity` and subclasses of `BaseThemedActivity` (e.g., OAuth login screens) did not call `SecureActivityDelegate.setSecure(this)` during their `onCreate` lifecycle method. This meant they bypassed the user's "Secure Screen" setting, potentially allowing sensitive crash logs or authentication flows to be captured in screenshots or the Android Recents app switcher.
**Learning:** Activities that do not inherit from the main application's base activity (`BaseActivity`), such as independent crash handlers or lightweight themed activities, require explicit setup for global security features like the `FLAG_SECURE` window attribute.
**Prevention:** Whenever creating new independent Activity entry points or base activity classes, explicitly call `SecureActivityDelegate.setSecure(this)` in their `onCreate` method to respect application-wide security settings.
