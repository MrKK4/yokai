## 2025-03-01 - Optimizing Collection Chains with Sequences in Tachiyomi/Yokai
**Learning:** In codebases dealing with potentially large datasets (like library manga lists or history), chained Kotlin collection operations (e.g., `.map { ... }.filter { ... }.take(10)`) create intermediate `ArrayList` allocations for each step. Given the memory-constrained Android environment, this can cause significant GC pressure and jank when searching.
**Action:** Always insert `.asSequence()` before chained collection operations on potentially large lists (e.g., manga library, catalogues), followed by a terminal operation like `.toList()`. This enforces lazy evaluation, avoiding unnecessary allocations and processing, which is especially critical for features like `UnifiedSearchPresenter` that search across multiple large sources simultaneously.
